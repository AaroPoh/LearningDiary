4.2 PreProsessointi funktio tehtävä

Tehtävänanto:

Tee funktio, jolle voi antaa 5 argumenttia: data-kehikon, muuttujat joita ei käännetä, muuttujat jotka käännetään (reverse coding), muuttujien minimin ja maksimin.

Funktio sitten muuntaa asteikon ulkopuolella olevat arvot puuttuviksi havainnoiksi, laskee summamuuttujan, tarkistaa poikkeavat havainnot, tekee tilastotestin summamuuttujan normaalijakautuneisuudelle, raportoi sen tuloksen selväsanaisesti (onko vai ei, mikä oli tulos), piirtää kuvan jossa kolme kuvaa vierekkäin: summamuuttujan histogrammi, summamuuttujan Q-Q plotin, summammuuttujaa vastaavan simuloidun normaalijakauman Q-Q plotin.

Sovella funktiota HSQ-aineistoon (Humor Styles Questionnaire)

palauta 2 tiedostoa, R-skripti ja pdf, jossa neljän HSQ-summamuuttujan jakaumat



Tehtävän ratkaisu:

Näytä koodi 
#ladataan data 
library(readr)
library(dplyr)
library(ggplot2)
library(patchwork)
HSQdata <- read_csv("HSQdata.csv")

# HSQ käännettävät muuttujat 'Q1', 'Q7', 'Q9', 'Q15', 'Q16', 'Q17', 'Q22', 'Q23', 'Q25', 'Q29', 'Q31, 


affiliative <- c("Q1", "Q5", "Q9", "Q13", "Q17", "Q21", "Q25", "Q29")

selfenhancing <- c("Q2", "Q6", "Q10","Q14", "Q18", "Q22","Q26", "Q30")

aggressive <- c("Q3", "Q7", "Q11", "Q15", "Q19", "Q23", "Q27", "Q31") 

selfdefeating <- c("Q4", "Q8", "Q12", "Q16", "Q20", "Q24", "Q28", "Q32")



PreProsFunktio<- function(data, variables = NULL, r_variables = NULL,  min = 1, max = 5, SumMuut, Otsikko = deparse(substitute(SumMuut))){
  
  # r_variables kohtaan sarakkeet, jotka käännetään
  # variables kohtaan muut 
  
  
  # Tarkistetaan, onko r_variables annettu ja onko siinä sisältöä
  if (!is.null(r_variables) && length(r_variables) > 0){
    
    data[, r_variables] <- (max + min) - data[, r_variables]
    message("Käänteiset muuttujat käännetty.")
  } else {
    message("Ei käännettäviä muuttujia, ohitetaan kääntö.")
  }
  
    
  
  #merkataan asteikon ulkopuoliset arvot puuttuviksi 
  # katsotaan mitkä arvot ovat liian pieniä TAI liian suuria
  virheelliset <- data < min | data > max
  
  # Korvataan nämä kohdat NA-arvolla
  data[virheelliset] <- NA
  message("Asteikon ulkopuoliset arvot poistettu")
  # tarkistetaan poikkeavat havainnot MAD-kriteerillä 
  
  # luodaan datafreimistä vektori, johon voidaan käyttääm mad() ja median() funktioita
  
  numarvot <- as.numeric(unlist(data[ ,1:32]))
  
  
  mad_value <- mad(na.omit(numarvot)) 
  median_value <- median(na.omit(numarvot))
  
  #Jos poikkeava havainto niin ilmoitetaan löytyi poikkeavia havaintoja, jos ei niin päinvastoin
  
  
  threshold <- 3 * mad_value
  
  filtered_data <- data[abs(data - median_value) >= threshold]
  
  if(length(filtered_data) > 0){
    message("Löytyi poikkeavia arvoja")
  } else {
    message("Ei poikkeavia arvoja")}
  
  #tehdään summamuuttujat eri huumorityypeille
  
  SummaMuuttuja <- rowMeans(data[ ,SumMuut], na.rm = TRUE)
  
  data$SummaMuuttuja <- SummaMuuttuja
  
ntesti <- shapiro.test(na.omit(SummaMuuttuja))

p_arvo <- as.numeric(ntesti) 

if(p_arvo[1] < 0.05){
  message("Shapiro-wilk testin p < 0.05, Summamuuttuja ei siis ole normaalisti jakautunut")
}else {
  message("Shapiro-wilk testin p > 0.05, Summamuuttuja on normaalisti jakautunut")
}

# Summamuuttujaa vastaava simuloitu normaalijakauma 

set.seed(123)
SimuloituJakauma <- rnorm(length(na.omit(SummaMuuttuja)), mean(SummaMuuttuja, na.rm = TRUE), sd(SummaMuuttuja, na.rm = TRUE))
  
Simdata <- data.frame(SimuloituJakauma)

# luodaan muuuttuja, johon tulee vain annetun summamuuttujan nimi
Labs <- deparse(substitute(SumMuut))
  
# Piirretään kuvaajat ggplotilla, qqplot, simuloidun jakauman qqplot ja histogrammi 
 p1 <- ggplot(data, aes(sample = SummaMuuttuja )) +
    geom_qq() +
    geom_qq_line(color = "red") +
    labs(title = paste(Otsikko, "QQ-plot")) +
    theme_minimal()
 
 p2 <- ggplot(Simdata, aes( sample = SimuloituJakauma )) + 
   geom_qq() +
   geom_qq_line(color = "red") +
   labs(title = "Simuloidun jakauman QQ-plot") +
   theme_minimal()
 
  
  
 p3 <- ggplot(data, aes( x = SummaMuuttuja)) + 
           geom_histogram(fill = "lightblue", color = "black", bins = 10) +
           labs(title = paste(Otsikko, "jakauma"), x = "Keskiarvo", y = "Frekvenssi") +
           theme_minimal()
  print(p1 + p2 + p3)
  
}




PreProsFunktio(HSQdata,  , c("Q1", "Q7", "Q9", "Q15", "Q16", "Q17", "Q22", "Q23", "Q25", "Q29", "Q31"), 1, 5, selfenhancing)


#Piirretään kuvaajat yhteeen pdf tiedostoon 

# luodaan lista, johon on tallennettu summamuuttujat 
kaikki_tyylit <- list(
  affiliative = affiliative, 
  selfenhancing = selfenhancing, 
  aggressive = aggressive, 
  selfdefeating = selfdefeating)

# while komento varmistaa, ettei pdf:n kirjoitus ole jäänyt auki aiemmin 
while (!is.null(dev.list())) dev.off()

#pdf komento avaa pdf:n kirjoituksen 
pdf("Summamuuttujien_kuvaajat.pdf", width = 14, height = 5)

try({

  # napataan listasta yksi huumorityyli kerrallaan
  nykyinen_setti <- kaikki_tyylit[[i]]

unlist()

  • muuntaa datan matriisi tai dataframe muodosta takaisin vektoriksi

  • esim.

 numarvot <- as.numeric(unlist(data[ ,1:32]))

  • unlist() komennolla pois kehikkomuodosta ja as.numeric() komennolla varmistetaan, että tallentuu numeerisena

  • numarvot <- as.numeric(unlist(data[ ,1:32]))



! huutomerkki tarkoittaa loogista ei operaattoria, && ja if lausekkeessa

  • esim.

    if (!is.null(r_variables) && length(r_variables) > 0)

  • huutomerkki tarkoittaa komennossa !is.null(), sitä että kun r_variables muuttuja ei ole tyhjä niin ehto täyttyy –> ts. is not null

  • && merkki tarkoittaa JA eli sitä, että molempien ehtojen täytyy täyttyä, jotta if lausekkeen ehdot toteutuvat

    • kahta && merkkiä käytetään lähinnä if lausekkeissa, muualla yhtä & merkki

  • Tai puolestaan merkitään | pystyviivalla, if lausekkeessa kannattaa käyttää kahta || ja muualla yksi | riittää

    • eli merkitsee if lausekkeessa sitä, että jomman kumman ehdon täyttyminen riittää



4.3 Visualisointitehtävä

Tehtävänanto :

Oheisessa data-setissä on tutkittu lääketieteen opiskelijoiden visuaalisen asiantuntijuuden (professional vision/perceptual learning) kehittymistä osana dermatologian kurssia. Diagnoosien erottelu-kokeet tehtiin kurssin aluksi, keskellä ja lopuksi (Session:1-3, pre-mid-post). Oppimista mitattiin kolmella muuttujalla: %-oikein, reaktioaika, confidence-rating,

Tee R-skripti, joka visualisoi tulokset, eli oppimisen/muutoksen kurssin aikana (pre-mid-post) kolmen outcome muuttujan suhteen.

Tutki dataa lisää, kokeessa oli 47 eri diagnoosia. Mitkä diagnoosit käyttäytyvät samanlaisesti? Klusteroi diagnooseja yhteen ja visualisoi oppiminen. Missä diagnooseissa opiskelijat tekivät eniten virheitä? Luokittele vaikeat/helpot diagnoosit ja visualisoi oppiminen. Ideoita/esimerkkejä voi katsoa artikkeleista (data osittain eri) tai keksiä ihan uusia tapoja.

Palauta R-skripti ja erikseen kuvat pdf:nä.
